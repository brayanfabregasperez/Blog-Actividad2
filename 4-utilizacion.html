Actividad 2: Tutorial sobre Angular.

4. Utilización del framework

a) ¿Cómo empezar a crear una aplicación base?

La forma recomendada es usar Angular CLI para crear la estructura del proyecto y el entorno de desarrollo. Angular CLI genera un scaffold completo con TypeScript, tests y configuración de build.

Para un inicio rápido, puedes usar Angular con CDN para pruebas simples, pero para proyectos reales es mejor usar la CLI. También puedes combinar ambos enfoques según tus necesidades.

Ejemplo: crear una aplicación base con Angular CLI

npm install -g @angular/cli
ng new mi-app
cd mi-app
ng serve
# Abrir http://localhost:4200

b) ¿Cómo visualizar resultados?

Con Angular CLI usa ng serve y abre http://localhost:4200. Los cambios se actualizan automáticamente con HMR/Live reload.

Para producción usa ng build --prod y sirve los archivos estáticos resultantes desde cualquier servidor estático.

También puedes usar herramientas como Vite o Webpack para servir y optimizar tu aplicación según tus necesidades.

c) ¿Cómo depurar errores?

Usa las DevTools del navegador (F12) para ver la consola. Para Angular hay herramientas como Angular DevTools (extensión) que te permiten inspeccionar componentes, perfiles y detectar problemas de rendimiento.

Además, Angular CLI muestra errores de compilación en la terminal donde ejecutas ng serve, facilitando la identificación y corrección de errores.
