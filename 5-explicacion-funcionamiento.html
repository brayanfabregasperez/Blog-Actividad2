Actividad 2: Tutorial sobre Angular.

5. Explicación del funcionamiento de Angular

Angular es un framework completo basado en componentes escritos en TypeScript. Usa un motor de renderizado y un sistema de enlace de datos (data binding) para sincronizar la vista y el modelo.

Angular utiliza un sistema de change detection para actualizar la interfaz cuando cambian los datos — combinado con el uso de RxJS para programación reactiva en la gestión de flujos y eventos.

a) ¿Actúa sobre páginas, ficheros, secciones...?

Angular actúa sobre el DOM vía plantillas (templates) con binding y directivas. Los componentes se combinan en módulos y el motor de Angular renderiza y actualiza las partes necesarias de la UI.

El enfoque modular y el sistema de inyección de dependencias permiten organizar y reutilizar código con claridad en aplicaciones grandes.

b) ¿Cómo se editan/modifican los ficheros?

Se editan archivos TypeScript (.ts) y plantillas HTML (.html); Angular CLI y herramientas como Webpack/Vite (si se usa) recompilan y recargan la app en desarrollo.

Angular fomenta separar lógica (component.ts), vista (component.html) y estilos (component.css/scss) para mantener el proyecto organizado.

c) ¿Cómo se referencian ficheros creados desde otras aplicaciones?

Angular consume APIs REST/GraphQL mediante servicios y HttpClient; se recomienda centralizar la lógica de comunicación en servicios inyectables.

También se pueden importar librerías externas y exponer componentes como web components para interoperar con otras tecnologías.
