Actividad 2: Tutorial sobre Angular.

2. Instalación

Angular normalmente se utiliza con herramientas oficiales (Angular CLI) y TypeScript. El flujo habitual es crear proyectos con la CLI para desarrollo y despliegue, lo que facilita generación de componentes, pruebas y optimización.

a) ¿Se precisa instalación?

Sí. Para usar Angular CLI necesitas Node.js y npm instalados en tu sistema, por si no los tienes aquí te dejamos los enlaces para descargarlos e instalarlos Node.js y npm. Angular CLI se instala globalmente con npm y luego se usa para crear y gestionar proyectos Angular.

También puedes usar Angular con CDN para pruebas rápidas, pero no es el enfoque recomendado para desarrollo serio.

b) Instalación de Node.js y npm

Para instalar Node.js y npm, sigue estos pasos:

  1. Visita la página oficial de Node.js en https://nodejs.org/.
  2. Descarga el instalador adecuado para tu sistema operativo (Windows, macOS, Linux).
  3. Ejecuta el instalador y sigue las instrucciones en pantalla. Asegúrate de seleccionar la opción para instalar npm junto con Node.js.
  4. Una vez completada la instalación, abre una terminal o línea de comandos y verifica que Node.js y npm se hayan instalado correctamente ejecutando los siguientes comandos: 
    node -v
npm -v

c) Pasos para la instalación (Angular CLI)

npm install -g @angular/cli
  ng new my-app
  cd my-app
  ng serve
  # Abrir http://localhost:4200

d) Versiones necesarias

Asegúrate de tener una versión LTS de Node.js compatible con Angular CLI (consulta la documentación de Angular para la versión recomendada). npm viene normalmente con Node.js y es necesario para instalar dependencias.

Angular CLI se actualiza regularmente, por lo que es recomendable mantenerlo actualizado con npm update -g @angular/cli. También es importante mantener actualizadas las dependencias del proyecto con npm update.

e) Rutas y variables

Angular gestiona su configuración en `angular.json` y las variables de entorno en `src/environments/` (p. ej. environment.ts). Los scripts útiles (ng serve, ng build) están en `package.json`. Así puedes personalizar el comportamiento del proyecto y gestionar diferentes entornos (desarrollo, producción).

Por ejemplo, puedes definir variables específicas para cada entorno y usar diferentes configuraciones de build.

f) ¿Necesita base de datos?

No. Angular es para la capa cliente; la persistencia se realiza en back-end (APIs/servicios). Angular incluye `HttpClient` para consumir APIs y manejar solicitudes HTTP de forma segura.

Se recomienda encapsular llamadas a APIs en servicios y gestionar autorización, errores y cache en capas específicas del proyecto.
